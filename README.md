# 打ち込みゼロ日報 — 現場日報 自動生成ツール

**塗装職人歴27年の実体験から設計した、「文字を打たない」現場日報アプリ。**
タップと音声だけで、30秒で日報が完成します。

🔗 **デモ（ブラウザでそのまま動きます）:** https://isao1301111-eng.github.io/genba-nippo-tool/

## なぜ作ったか

現場日報の最大のネックは**文字の打ち込み**です。職人はスマホで長文を打ちません。
また、日報の「進捗40%」は**何に対する40%か分からず**、引き継ぎ事故（二度塗り・工程飛ばし）や、翌日の材料の買い忘れによる**現場ストップ**が起きます。

このツールは、27年間現場で感じてきたその課題を、AI設計の考え方で解決します。

## 主な機能（すべて現場目線）

| 機能 | 現場の課題 |
|------|-----------|
| **打ち込みゼロ入力** — 現場・天候・工程・材料・人数すべてタップ、特記事項は音声入力 | 職人は文字を打たない |
| **出来高ベースの全体進捗％** — 工程ごとの状況（7段階）から自動算出 | 「40%」が何の40%か分からない問題 |
| **人工（にんく）自動計算** — 人数×作業時間から本日・累計人工を自動集計 | 職人の原価感覚は「人工」 |
| **引き継ぎ欄** — 現場を選ぶだけで、どの工程が・どの箇所まで終わっているか一目で分かる | 別の職人が入るときの引き継ぎ事故 |
| **翌日/次回に必要な塗料・材料** — 日報がそのまま発注リストに | 欠品による現場ストップ |
| **工程・箇所のカスタム** — 現場に合わせて追加/削除、保存して次回も使える | 足場や洗浄がない現場もある |
| **現場一覧の管理** — 直近5件だけ表示、🏷タグ付け・検索・並び替え（直近順/名前順/人工順） | 現場が増えると探せない |

## 設計思想：ハルシネーション防止

見積・日報の数字は1桁違うだけで赤字や事故につながります。
本ツールは**入力・選択された情報のみ**で日報を作成し、未入力の項目はAIに推測させず**「（記載なし）」と明示**します。

同じ思想で設計した見積ツール: [samurai-painting-quote-tool](https://github.com/isao1301111-eng/samurai-painting-quote-tool)

## 技術構成

- 単一HTMLファイル（依存ライブラリなし・APIキー不要・オフライン動作）
- 音声入力: Web Speech API（Chrome/Edge）
- 写真の撮影日時: EXIF自動抽出（自前パーサ実装）
- データ保存: localStorage（現場ごとの進捗・人工・工程状況・タグを記憶）
