# 通知・連携（GAS）セットアップガイド

日報の作成・更新時に、**①代表者・経理へメール通知 ②Googleスプレッドシートへ自動記録**を行う設定です。
所要時間：約10〜15分。外部サービスは使わず、すべて**あなた自身のGoogleアカウント内**で完結します。

---

## 全体の流れ
```
Googleスプレッドシート作成 → Apps Scriptにコード貼付 → ウェブアプリとして公開
 → 発行されたURLをアプリの「通知・連携設定」に貼る → テスト送信で確認
```

---

## 手順

### 1. スプレッドシートを作る
1. https://sheets.google.com を開き、空白のスプレッドシートを新規作成
2. 名前を「**現場日報ログ**」などに変更（任意）

### 2. Apps Scriptを開く
1. メニューの **拡張機能 → Apps Script** をクリック
2. 開いたエディタの既存コード（`function myFunction(){}`）を**全部消す**

### 3. コードを貼り付ける
1. 同じフォルダの **`通知連携_GASコード.gs`** の中身を全部コピー
2. Apps Scriptエディタに貼り付けて、**💾 保存**（Ctrl+S）

### 4. ウェブアプリとして公開する
1. 右上の **デプロイ → 新しいデプロイ**
2. 歯車アイコン → **ウェブアプリ** を選択
3. 設定：
   - 説明：任意（例「日報通知」）
   - **次のユーザーとして実行：自分**
   - **アクセスできるユーザー：全員**  ← ここ重要
4. **デプロイ** をクリック
5. 初回は「承認が必要」と出る → **アクセスを承認**
   - 「このアプリは確認されていません」と警告が出たら
     **詳細 → （プロジェクト名）に移動（安全ではないページ）** → **許可**
   - ※これはあなた自身が作ったアプリなので安全です
6. 表示された **ウェブアプリのURL**（`https://script.google.com/macros/s/......./exec`）を**コピー**

> 動作確認：そのURLをブラウザで開いて「OK - genba-nippo notify endpoint is running」と出れば公開成功。

### 5. アプリに設定する
1. 「打ち込みゼロ日報」を開き、一番下の **⚙ 通知・連携設定** へ
2. **ウェブアプリURL** に手順4のURLを貼る
3. **通知先メール** に代表者・経理のアドレスをカンマ区切りで入力
   （例：`daihyo@example.com, keiri@example.com`）
4. **「日報の作成・更新時に自動で通知＋記録する」にチェック**
5. **💾 保存** → **✉ テスト送信**
6. 指定メールにテスト通知が届き、スプレッドシートに1行増えていれば**完成**

---

## これで実現すること
- 職人が日報を作成/更新するたびに、**代表者・経理へ自動でメール**が届く
- 全日報が**1つのスプレッドシートに自動蓄積**（会社の台帳が勝手にできる）
- 記録には**作成者・作成日時・更新者・IP・人工・進捗**が含まれる

## 補足・注意
- **コードを変更したら**：デプロイ → デプロイを管理 → 鉛筆マーク → バージョン「新バージョン」→ デプロイ（URLは変わりません）
- **メールが届かない**：通知先アドレスの綴り、迷惑メールフォルダを確認
- **Gmailの送信上限**：無料アカウントは1日約100通まで（通常の現場運用なら十分）
- URLは第三者に教えないでください（知っている人はデータを送信できてしまうため）
